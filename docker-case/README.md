# Инфраструктура микросервисного приложения: Multi-Vendor E-Shop

Проект представляет собой реализацию среды для запуска стека MERN (MongoDB, Express, React, Node.js) с использованием Docker Compose. Основная цель проекта — автоматизация развертывания, изоляция компонентов и оптимизация Docker-образов для использования в CI/CD пайплайнах.

## Архитектура решения

Инфраструктура разделена на логические слои, каждый из которых работает в изолированном контейнере:

1. **Слой представления (Frontend):** Статическое React-приложение.
2. **Слой бизнес-логики (Backend):** REST API сервис на базе Node.js.
3. **Слой обмена данными (Socket):** Сервер для обработки событий в реальном времени.
4. **Слой данных (Database):** NoSQL база данных MongoDB.

## Ключевые инфраструктурные решения


### Оркестрация и безопасность сети
Все микросервисы объединены в изолированную bridge-сеть Docker. 
* База данных, Socket-сервер и Backend не имеют прямого доступа извне (порты не проброшены на хост-машину без необходимости).
* Доступ пользователей ограничен только 3000 портом веб-сервера.
* Настроены именованные тома (Docker Volumes) для обеспечения персистентности данных MongoDB при пересоздании контейнеров.

## Инструкция по развертыванию

### Системные требования
* Docker Engine 20.10+
* Docker Compose v2.0+

### Запуск окружения
1. Клонируйте репозиторий:
   ```bash
   git clone <repository_url>
   cd <project_directory>
   ```

2. Подготовьте переменные окружения (создайте файл на основе примера):
   ```bash
   cp .env.example .env
   ```

3. Запустите инфраструктуру в фоновом режиме:
   ```bash
   docker-compose up -d --build
   ```

## Мониторинг и обслуживание

Проверка статуса запущенных контейнеров и утилизации ресурсов:
```bash
docker-compose ps
docker stats
```

Просмотр потока логов бэкенд-сервиса:
```bash
docker-compose logs -f backend
```
